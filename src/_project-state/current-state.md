## Adaptive engine + scoring + norms (`src/features/assessment/`, `src/features/scoring/`, `src/content/norms/`) — Phase 1.05, **calibration v2 in 2.06**

The **brain** of the assessment: a pure, deterministic, UI-free state machine + scoring layer that turns a child's responses into the five parent-facing indices with bands, confidence, validity, and extremes. No clock, no randomness beyond the seeded PRNG, no React — same `sessionSeed` + age + response/timing script → **deep-equal `AssessmentResult`**, always (purity- and determinism-tested, mirroring 1.04).

- **`content/norms/seed-norms.ts`** — the **single tuning surface**, now **calibration v2** (filename kept to avoid import churn): per-SIGNAL start-level tables (`START_LEVELS`), Corsi span expectations (fractional, backward offset 0.5), v2 item caps (lone 5/6/7 · shared 4/5/6 per age cluster), age-banded attention/validity thresholds (`ATTENTION_BANDS`), per-age index anchors (`expectedWeightedAccuracy` + the Gs throughput table), idle/confidence thresholds and composite weights — plus the **`PROVISIONAL_NORMS` register** (every interpolated value + reason, completeness-tested) as the Phase-3 pilot worklist. `SCORING_VERSION` + `NORMS_VERSION` = `2.0.0`, carried in `result.meta` with `normsStage: "seed"` (still pre-pilot).
- **`features/assessment/`** — `startSession` → `nextAction` (selector) → `applyResponse` (reducer) → `advanceDomain`, plus a `runSession` driver. **Two control flows (v2)** behind one interface: laddered basal/ceiling for Gf, Gv, Gsm, EF, Glr and CT — per-signal starts, the **WISC basal reverse rule** (a wrong FIRST item demotes item-by-item with the ceiling suspended; levels below the first correct are credited at their level weight), 2-consecutive-error ceiling + age item caps, Corsi rows carrying length/direction/path (under-8 backward→forward+crisscross substitution, 6-/9-tile boards), Glr rows carrying pairs/trials — and the fixed speeded exception Gs (per-age parameters, 2 scored rounds over the same targets, no staircase/basal). Per-item seeds via `deriveSeed`; each administered item comes from `generateItem` (age-aware). Plus **`fixtures.ts`** — five reusable scripted profiles (logic-strong / spatial-strong / flat / ceiling / strong-invalid), reused by 1.07.
